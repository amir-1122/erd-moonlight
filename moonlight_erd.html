🔄 Core Business Flows
🏥 Facility Workflow
1
users.user_type = 'facility' created → email OTP sent
otp_verifications.purpose = 'registration'
2
Facility completes profile → facilities.profile_status = 'complete'
3
Admin approves → facilities.verification_status = 'approved'
4
Creates shift → shifts.status = 'draft'
Can duplicate: parent_shift_id populated on copies
5
Publishes shift → shifts.status = 'published'
6
Provider joins → shifts.status = 'filled' automatically
7
Confirms completion → shift_assignments.facility_confirmed_at set
8
Monthly invoice generated → invoices + invoice_line_items
👨‍⚕️ Provider Workflow
1
users.user_type = 'provider' created → email OTP sent
2
Sees limited shifts (incomplete profile filter applied in API)
providers.profile_status = 'incomplete'
3
Uploads credentials → provider_credentials rows created
4
Admin approves all → providers.verification_status = 'approved'
5
Registers for shift → shift_assignments row auto-created
Blacklist check runs first: provider_blacklist query
6
Overlap check: no two assignments with overlapping times
7
Submits review → shift_reviews (reviewer_type='provider')
8
Payment initiated → payments row, net_amount to provider
📅 Shift Assignment Logic
1
Provider swipes up on shift detail → "Join" intent fired
2
GUARD: providers.verification_status = 'approved'?
No → redirect to profile completion
3
GUARD: provider_blacklist check for this facility
Blocked → shift never shown / error returned
4
GUARD: overlap check on shift_assignments for same provider
Conflict shown: "You already have shift X at this time"
5
GUARD: shifts.status = 'published' and no existing assignment
6
All pass → shift_assignments INSERT, shifts.status → 'filled'
Atomic transaction. No facility approval needed.
💰 Payment Trigger Logic
1
shift_assignments.status = 'completed' set
2
shift_reviews submitted by provider (reviewer_type='provider')
3
Facility confirms: shift_assignments.facility_confirmed_at set
4
Steps 2+3 done → payments row created (status='pending')
Independent of facility paying Moonlight
5
Moonlight processes payout → payments.status = 'completed'
6
Facility billed monthly: invoices + invoice_line_items generated
